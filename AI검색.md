# 네이버 검색 API 활용: 당일 급등주 및 거래대금 상위 종목 상승 사유 정밀 추적 가이드

이 문서는 네이버 검색 API(뉴스, 블로그 등)를 활용하여 **"특정 종목이 오늘 왜 급등했는지"**, **"왜 거래대금이 몰렸는지"**에 대한 핵심 사유와 모멘텀을 가장 정확하게 추출하기 위한 프롬프트 엔지니어링 및 쿼리 최적화 가이드라인입니다. (`뉴스분석.py`가 쓰는 패턴 기준)

LLM(클로드 등) 분석용 AI 에이전트 개발이나 데이터 수집 파이프라인 구축 시 참조할 수 있도록 검색 쿼리 설계 패턴, API 파라미터 튜닝, 노이즈 필터링 전략을 담고 있습니다.

---

## 1. 네이버 검색 API의 특성 이해
네이버 검색 API는 LLM과 같은 생성형 AI가 아니라 **키워드 매칭 기반의 검색 엔진**입니다. 따라서 문장형 질문(`"오늘 삼성전자가 왜 이렇게 많이 올랐어?"`)보다 언론사, 애널리스트, 투자자들이 **글을 작성할 때 주로 사용하는 전문 키워드와 연산자의 조합**으로 접근해야 가장 정확한 결과를 얻을 수 있습니다.

---

## 2. 목적별 최적의 검색 쿼리(Query) 패턴

종목명을 변수(`{stock_name}`)로 처리하여 API 요청(`query`) 파라미터에 전달할 때, 아래의 패턴들을 조합하여 다중 검색(Multi-query)을 수행하는 것이 가장 효과적입니다.

### ① [필수] 당일 급등 및 테마성 재료 포착 (뉴스 탭 타겟)
언론사에서 당일 주가 급등이나 거래량 폭발이 일어난 종목의 사유를 보도할 때 무조건 사용하는 정형화된 키워드입니다.
* `"{stock_name} 특징주"` (★ 가장 정확도 높음. 당일 호재 기사 강제 타겟팅)
* `"{stock_name} 급등 이유"` 또는 `"{stock_name} 상승 배경"`
* `"{stock_name} 상한가 사유"`

### ② [선택] 대규모 수급 및 거래대금 집중 사유 포착 (뉴스/블로그 혼합)
단순 주가 상승 외에 거래대금이 폭발한 원인(공시, 대규모 수주, 글로벌 트렌드 편입 등)을 추적할 때 유용합니다.
* `"{stock_name} 거래량 폭발"`
* `"{stock_name} 모멘텀"`
* `"{stock_name} 공급계약 공시"`
* `"{stock_name} 대규모 수주"`

### ③ API용 검색 연산자(Boolean Operator) 활용 패턴
네이버 API는 기본적인 연산자를 지원합니다. 공백은 기본적으로 `AND` 조건을 의미하며, 반드시 포함되어야 하는 핵심 단어는 기호나 공백으로 묶어 정확도를 제어합니다.
* `"{stock_name} 특징주 상승"` (종목명과 특징주, 상승이 모두 포함된 문서)
* `"{stock_name} (급등|폭등|상승) 배경"` (급등, 폭등, 상승 중 하나와 종목명, 배경이 매칭)

## 3. 정확도를 극대화하는 API 파라미터(Parameter) 설정

단순히 키워드만 잘 넣는 것으로는 부족합니다. '오늘' 왜 올랐는지를 찾기 위해서는 API 호출 시 파라미터를 정밀하게 조정해야 합니다.

| 파라미터명 | 추천 설정 값 | 이유 및 설명 |
| :--- | :--- | :--- |
| **`sort`** | **`date`** (날짜순) | 기본값인 관련도순(`sim`)으로 설정하면 수개월~수년 전 과거의 급등 기사나 분석 글이 상위에 노출됩니다. **반드시 날짜순으로 정렬**하여 오늘 발생한 신규 뉴스/포스팅을 수집해야 합니다. |
| **`display`** | **`10` ~ `20`** | 당일 급등 사유는 당일 발행된 상위 10~20개의 뉴스 기사 헤드라인 및 본문 초록(Snippet)만으로도 95% 이상 파악이 가능합니다. 너무 많이 호출하면 노이즈가 섞입니다. |
| **`start`** | **`1`** | 최신 데이터를 가져와야 하므로 첫 페이지부터 탐색합니다. |

## 4. 날짜 기반 검색 범위 전략

네이버 뉴스 API는 날짜 범위 파라미터를 지원하지 않으므로, 응답 결과의 `pubDate` 필드를 코드 단에서 필터링한다.

| 상황 | 검색 범위 | 설명 |
| :--- | :--- | :--- |
| **당일 또는 최근 1~2일** | 파일날짜 ~ 오늘 | 장 마감 직후 실행 시, 당일 포함 최신 기사까지 수집 |
| **파일날짜 기준 3일 이상 경과** | 파일날짜 ~ 파일날짜+3일 | 과거 날짜 재분석 시 범위를 제한하여 해당 시점 뉴스만 수집 |

**우선순위 정렬**: 날짜 필터 통과 후 `pubDate`가 파일날짜에 가까울수록 우선 배치 (당일 기사 최우선).

```python
# 검색 범위 계산 예시
file_date  = "2026-06-17"
today      = "2026-06-18"  # 1일 경과 → end_date = today
# today    = "2026-07-01"  # 14일 경과 → end_date = file_date + 3일 = "2026-06-20"
```

## 5. 정보 노이즈 및 스팸 필터링 전략 (Data Cleansing)

주식 관련 키워드는 네이버 검색 시 자동 생성된 광고성 블로그 글, 카카오톡 리딩방 가입 유도 스팸 글이 대거 유입됩니다. LLM(클로드)에 데이터를 넘겨주기 전, 다음과 같은 불필요한 단어가 포함된 검색 결과(Snippet)는 코드 단에서 1차 필터링하는 것이 좋습니다.

* **제외 키워드 리스트:**
  `"무료 리딩방"`, `"카톡방"`, `"클릭 시 이동"`, `"급등주 추천 받기"`, `"체험 신청"`, `"VIP 회원"`, `"선착순 모집"`
* **API 쿼리 레벨에서의 제외:**
  네이버 API 자체에서 마이너스(`-`) 연산자를 지원하므로 아래와 같이 쿼리를 구성하여 스팸을 차단할 수 있습니다.
  > 쿼리 예시: `"{stock_name} 특징주 -리딩방 -카톡방 -추천"`

## 6. 시스템 워크플로우 구성 추천 (For Claude AI Agent)

클로드(Claude) 등 LLM 기반 에이전트에게 데이터를 전달하여 최종 리포트를 생성할 때 권장하는 파이프라인입니다.

1. **1단계: 데이터 수집 (API 호출)**
   * 장 마감 후 거래대금 상위 종목 리스트를 추출합니다.
   * 각 종목에 대해 §2의 필수+선택 쿼리 패턴으로 **네이버 뉴스 API(sort=date)**를 호출합니다.
   * §4 날짜 전략에 따라 파일날짜 기준으로 범위를 제한하고 당일 기사를 우선 정렬합니다.
2. **2단계: 클로드 프롬프트 주입**
   * 수집된 가공 데이터(뉴스 제목, 요약문)를 클로드에게 컨텍스트로 제공하며 요약을 요청합니다.
