# 기능 개요

매일 장마감 후 **Python 스크립트**가 FinanceDataReader로 KRX 전종목 시세를 자동 수집해
MongoDB에 저장 → 웹앱 달력에서 날짜 클릭 시 데이터·AI 분석 결과 자동 표시.
HTS 엑셀 수동 업로드 단계 없이 Python 스크립트 + MongoDB 기반으로 완전 자동화됨.
거래대금·등락률 상위 50은 FDR(KRX 단독) 후 KIS 통합(KRX+NXT, `FID_COND_MRKT_DIV_CODE=UN`)
데이터로 다시 보강함(2026-06-21) — NXT(대체거래소)에서 체결된 거래량·거래대금이 빠지지 않게.
종목 클릭 시 KIS(한국투자증권) Open API로 일봉·주봉 차트(이동평균선 포함)를 실시간 조회.
KIS 장애 시에만 토스증권 캔들 캐시(일봉만)로 폴백. 차트는 캔들이 아니라 OHLC 바 모양으로 표시됨.

## 달력·주차(W##)

달력에는 토·일 없이 월~금만 표시되고, 그 대신 각 행 맨 끝에 주차(W##) 칸이 있어 누르면
그 주(월~금) 코스피·코스닥 변동률을 보여준다(2026-06-22, 그 전에는 토요일 칸이었음).
`python 주간분석.py`가 FDR로 미리 계산해 MongoDB에 저장해두고 웹앱은 그 결과만 읽음
(실시간 KIS 호출 아님). **주차 칸을 누르면 코스피·코스닥 변동률과 함께 그 주(휴장일 제외)
거래대금·등락률 상위 50 표도 같이 보임**(2026-06-27 추가) — `뉴스분석.py`가 매일 하는
선정(FDR 전종목 + KIS 통합(KRX+NXT) 보강)을 한 주 단위로 적용한 것으로, 일간 표와 같은
컴포넌트를 그대로 재사용해 보여줌(단, 60일 신고가대비 열은 주간에서만 숨기고, 대신
"카테고리" 열을 등락률 열 옆에 추가로 보여줌 — 주간분석.py가 high60Rate는 계산 안 하지만
카테고리는 채워주기 때문, 2026-06-28). **거래대금·등락률 상위 50 카테고리 비중 도넛 차트도 주간에서
그대로 보여줌**(2026-06-27 추가) — 주간분석.py는 Claude/Codex를 호출하지 않아 직접
카테고리를 매기지 못하므로, 그 주에 해당하는 일간 분석들에서 이미 매겨진 카테고리를
종목명으로 찾아와 재사용함(그 주 어느 날의 일간 상위 50에도 없었던 종목은 "기타"로 표시).
일간 전용인 "거래대금·등락률 카테고리 TOP5 추이"(날짜별 추이표 2장)는 주간에는 없음. **`python
주간분석.py` 실행 후에는 Claude Code가 매번 자동으로 `weekly_indices`에서 카테고리가
전혀 없는 종목(진짜 미분류)만 찾아 직접 분류해 채워주는 게 표준 워크플로우**(2026-06-28
확정, 사용자 요청 — 별도로 매번 요청하지 않아도 됨). 이미 카테고리가 있는데 그 주
상위 5위 안에 못 들어 도넛 차트에서 "기타"로 합쳐 보이는 종목은 건드리지 않음 — 자세한
절차는 [DATA_PIPELINE.md](DATA_PIPELINE.md)의 "주간분석.py" 절 9번 참고.
**카테고리 비중 도넛과 주간 종목 데이터 표 사이에 "ETF 등락률 상위 15" 표도 같이 보임**
(2026-07-04 도입, 2026-07-06부터 이 위치로 — 원래 별도 "ETF 분석" 탭이었음) — 최소
순자산(AUM) 100억원 이상인 ETF만 대상으로 그 주(월~금) 등락률 상위 15개를 순위·현재가·
등락률·순자산 컬럼으로 보여준다(정렬 가능). `python 주간분석.py`가 vol/rate와 함께
같은 주기로 계산해두므로 별도 스크립트 실행이 필요 없고, 아직 그 주 데이터가 없으면
"이번 주 ETF 랭킹 데이터가 아직 없습니다" placeholder가 표시된다. ETF의 세부 업종/테마
분류(카테고리)는 표시하지 않음 — `FinanceDataReader`가 주는 ETF 분류 코드가 1~7의 내부
숫자일 뿐 공식 매핑표가 없어(직접 확인), 잘못된 라벨을 보여주는 대신 아예 생략함.
종목명으로 구성 ETF를 검색하는 기능은 있었으나 KIS 구성종목 API가 불안정해 실제로 정상
동작한 적이 없어 2026-07-06에 삭제함(자세한 경위는 [HISTORY.md](HISTORY.md) 참고).

## 카테고리 분류

분석 시 그날의 테마·종목을 24개 카테고리(반도체/반도체장비/조선/방산/AI/로봇 등, **평소엔 이름·정의만
수정 가능하고 추가·삭제는 안 함**)로 분류해서, 그날 거래대금/등락률 상위 50 각각의
카테고리별 종목 수 TOP5를 최근 영업일별로 나열한 표 2장("거래대금 카테고리 TOP5 추이",
2026-06-25 도입, 2026-06-28 꺾은선 그래프에서 표로 전면 교체 + 같은 날 바로 아래에
"등락률 카테고리 TOP5 추이" 표도 추가)과, 거래대금/등락률 각각의 그날 카테고리별 종목 수를
보여주는 도넛 차트 2장("거래대금·등락률 카테고리 비중", 가로 스와이프 전환, 2026-06-26)으로
보여줌.
거래대금·등락률 종목은 카테고리가 필수라 하나라도 빠지거나 목록 밖 값이면 `저장분석.py`가
MongoDB 저장 자체를 거부함(2026-06-27). 같은 종목이 다른 날엔 다른 카테고리로 분류돼도
의도된 동작 — 분석하는 그 시점에(여러 카테고리에 부합하면 그날의 `상승원인` 뉴스를 기준으로)
가장 부합하는 카테고리에 배치하면 되고 과거와 맞추지 않음. 기존 24개에 잘 안 맞는 종목은
정식으로 카테고리를 추가하기 전까지 `기타` + `신규카테고리후보` 필드로 표시해두며, 도넛
차트의 "기타" 범례를 펼치면 그런 종목의 이름이 빨간색으로 보임(2026-06-27).
카테고리 목록 값·분류 경계 기준·개편 이력은 [DATA_PIPELINE.md](DATA_PIPELINE.md)의
"`테마` 배열을 쓸 때 지킬 규칙" 절 참고.

## 탭 구성

화면 상단에는 탭 4개(거래대금·등락률 분석 / 종목 분석 / 차트분석 / 조건 검색)가
있고, 차트분석·조건 검색은 아직 내용 미정인 "준비 중" placeholder 상태. **"종목 분석" 탭은
2026-06-25부터 내용이 채워짐** —
기업개요·재무상태표·손익계산서 3개 버튼이 실제로 동작(기업개요: PER/PBR/ROE/EV·EBITDA + 목표
PER·PBR·EV·EBITDA·WACC를 슬라이더로 조절하는 적정주가 4종 / 재무상태표·손익계산서: CAPEX·
재고자산·매출채권·현금 비중, 부채비율, 매출액·영업이익·영업이익률·당기순이익·주당순이익 등 +
23·24·25년 연간과 올해 최근 분기까지 4개 기간 추이 그래프, 분기 값은 연환산 없이 실제값
그대로 표시). `python 종목분석.py 종목명`을 실행하면 MongoDB `company_analysis` 컬렉션에
저장되고, 탭 상단 검색창에 종목명을 입력(자동완성 지원)하면 3개 버튼 모두 바로 조회된다
(2026-06-25, 종목 검색 자동 연동). **기업개요의 현재가·시가총액·PER·PBR은 검색하는 순간
KIS Open API로 실시간 재조회**(2026-06-25 추가) — 종목분석.py를 실행한 날의 가격에 고정되지
않고 항상 그날그날의 실제 가격 기준으로 보임(KIS 호출 실패 시에만 저장된 값으로 폴백).
현금흐름표만 여전히 내용 미정 placeholder. **2026-06-27부터는 `python 종목분석.py 종목명`을
먼저 실행할 필요 없이, 검색창에 분석 안 된 종목명을 입력하면 서버가 그 자리에서 DART+KIS를
직접 호출해 즉석으로 분석하고 보여줌**(`api/analyzeCompany.js` — Claude/LLM 미사용, 전부
결정적 API 호출+산술 계산이라 사용량과 무관). 최초 1회만 DART 재무제표 수집 때문에 몇십 초
걸리고, 그다음부터는 기존 캐시 경로(KIS 실시간 재조회만)로 즉시 조회됨.
