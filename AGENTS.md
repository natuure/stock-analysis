# 주식 거래대금·등락률 분석 — 프로젝트 컨텍스트

## 프로젝트 개요
매일 장마감 후 **Python 스크립트**가 FinanceDataReader로 KRX 전종목 시세를 자동 수집해
MongoDB에 저장 → 웹앱 달력에서 날짜 클릭 시 데이터·AI 분석 결과 자동 표시.
HTS 엑셀 수동 업로드 단계 없이 Python 스크립트 + MongoDB 기반으로 완전 자동화됨.
거래대금·등락률 상위 50은 FDR(KRX 단독) 후 KIS 통합(KRX+NXT, `FID_COND_MRKT_DIV_CODE=UN`)
데이터로 다시 보강함(2026-06-21) — NXT(대체거래소)에서 체결된 거래량·거래대금이 빠지지 않게.
종목 클릭 시 KIS(한국투자증권) Open API로 일봉·주봉 차트(이동평균선 포함)를 실시간 조회.
KIS 장애 시에만 토스증권 캔들 캐시(일봉만)로 폴백. 차트는 캔들이 아니라 OHLC 바 모양으로 표시됨.
달력에는 토·일 없이 월~금만 표시되고, 그 대신 각 행 맨 끝에 주차(W##) 칸이 있어 누르면
그 주(월~금) 코스피·코스닥 변동률을 보여준다(2026-06-22, 그 전에는 토요일 칸이었음).
`python 주간분석.py`가 FDR로 미리 계산해 MongoDB에 저장해두고 웹앱은 그 결과만 읽음
(실시간 KIS 호출 아님). **주차 칸을 누르면 코스피·코스닥 변동률과 함께 그 주(휴장일 제외)
거래대금·등락률 상위 50 표도 같이 보임**(2026-06-27 추가) — `뉴스분석.py`가 매일 하는
선정(FDR 전종목 + KIS 통합(KRX+NXT) 보강)을 한 주 단위로 적용한 것으로, 일간 표와 같은
컴포넌트를 그대로 재사용해 보여줌(단, 60일 신고가대비 열은 주간에서만 숨김 — 주간분석.py가
이 값을 계산하지 않아서). **거래대금·등락률 상위 50 카테고리 비중 도넛 차트도 주간에서
그대로 보여줌**(2026-06-27 추가) — 주간분석.py는 Claude/Codex를 호출하지 않아 직접
카테고리를 매기지 못하므로, 그 주에 해당하는 일간 분석들에서 이미 매겨진 카테고리를
종목명으로 찾아와 재사용함(그 주 어느 날의 일간 상위 50에도 없었던 종목은 "기타"로 표시).
일간 전용인 "거래대금 카테고리 TOP5 추이"(날짜별 추이표)는 주간에는 없음.
분석 시 그날의 테마·종목을 19개 카테고리(반도체/반도체장비/조선/방산/AI/로봇 등, **평소엔 이름·정의만
수정 가능하고 추가·삭제는 안 함**)로 분류해서, 그날 거래대금 상위 50의 카테고리별 종목 수
TOP5를 최근 영업일별로 나열한 표("거래대금 카테고리 TOP5 추이", 2026-06-25 도입, 2026-06-28
꺾은선 그래프에서 표로 전면 교체)와, 거래대금/등락률 각각의 그날 카테고리별 종목 수를
보여주는 도넛 차트 2장("거래대금·등락률 카테고리 비중", 가로 스와이프 전환, 2026-06-26)으로
보여줌.
거래대금·등락률 종목은 카테고리가 필수라 하나라도 빠지거나 목록 밖 값이면 `저장분석.py`가
MongoDB 저장 자체를 거부함(2026-06-27). 같은 종목이 다른 날엔 다른 카테고리로 분류돼도
의도된 동작 — 분석하는 그 시점에(여러 카테고리에 부합하면 그날의 `상승원인` 뉴스를 기준으로)
가장 부합하는 카테고리에 배치하면 되고 과거와 맞추지 않음. 기존 19개에 잘 안 맞는 종목은
정식으로 카테고리를 추가하기 전까지 `기타` + `신규카테고리후보` 필드로 표시해두며, 도넛
차트의 "기타" 범례를 펼치면 그런 종목의 이름이 빨간색으로 보임(2026-06-27).
화면 상단에는 탭 3개(거래대금·등락률 분석 / 종목 분석 / 조건 검색)가 있고, 조건 검색은 아직
내용 미정인 "준비 중" placeholder 상태. **"종목 분석" 탭은 2026-06-25부터 내용이 채워짐** —
기업개요·재무상태표·손익계산서 3개 버튼이 실제로 동작(기업개요: PER/PBR/ROE/EV·EBITDA + 목표
PER·PBR·EV·EBITDA·WACC를 슬라이더로 조절하는 적정주가 4종 / 재무상태표·손익계산서: CAPEX·
재고자산·매출채권·현금 비중, 부채비율, 매출액·영업이익·영업이익률·당기순이익·주당순이익 등 +
23·24·25년 연간과 올해 최근 분기까지 4개 기간 추이 그래프, 분기 값은 연환산 없이 실제값
그대로 표시). `python 종목분석.py 종목명`을 실행하면 MongoDB `company_analysis` 컬렉션에
저장되고, 탭 상단 검색창에 종목명을 입력(자동완성 지원)하면 3개 버튼 모두 바로 조회된다
(2026-06-25, 종목 검색 자동 연동). **기업개요의 현재가·시가총액·PER·PBR은 검색하는 순간
KIS Open API로 실시간 재조회**(2026-06-25 추가) — 종목분석.py를 실행한 날의 가격에 고정되지
않고 항상 그날그날의 실제 가격 기준으로 보임(KIS 호출 실패 시에만 저장된 값으로 폴백).
현금흐름표만 여전히 내용 미정 placeholder. **2026-06-27부터는 `python 종목분석.py 종목명`을
먼저 실행할 필요 없이, 검색창에 분석 안 된 종목명을 입력하면 서버가 그 자리에서 DART+KIS를
직접 호출해 즉석으로 분석하고 보여줌**(`api/analyzeCompany.js` — Claude/LLM 미사용, 전부
결정적 API 호출+산술 계산이라 사용량과 무관). 최초 1회만 DART 재무제표 수집 때문에 몇십 초
걸리고, 그다음부터는 기존 캐시 경로(KIS 실시간 재조회만)로 즉시 조회됨.

이 파일은 목차 역할만 한다 — 세부 내용은 아래 표에서 안내하는 파일을 직접 열어서 참고할 것.

## 데이터 흐름 (요약)

```
python 뉴스분석.py  ← 장마감 후 실행
    ├── FinanceDataReader로 전종목 + 코스피/코스닥 지수 수집
    ├── MongoDB stock_data 컬렉션 저장
    ├── 토스증권 캔들 미리 조회 → MongoDB candles 컬렉션 캐싱 (KIS 장애 시 폴백용)
    ├── Naver 뉴스 API 검색
    └── 분석결과/뉴스데이터_YYYYMMDD.json 저장
         ↓
Codex "분석해줘" 요청 → 분석결과/분석결과_YYYY-MM-DD.json 생성
         ↓
python 저장분석.py → MongoDB ai_analysis 컬렉션 저장

python 주간분석.py  ← 아무 때나 실행 가능 (별도 흐름, 위 일간 분석과 무관)
    ├── FDR로 코스피/코스닥 주간(월~금) 변동률 계산
    └── 뉴스분석.py를 import해 KIS 통합(KRX+NXT) 보강 재사용 → 주간 거래대금·등락률
          상위 50도 같이 계산(2026-06-27 추가) → MongoDB weekly_indices 컬렉션 저장
         ↓
웹앱: /api/getData, /api/getAnalysis 는 MongoDB만 읽어서 화면 표시
     (달력 주차(W##) 칸의 주간 변동률은 /api/getData가 weekly_indices를 같이 읽어 내려주고,
     그 주차를 클릭하면 /api/getData?week=로 그 주의 거래대금·등락률 상위 50도 지연 조회함)
     /api/candles 는 KIS Open API를 직접 호출(실시간) → 실패 시에만 candles 캐시(토스) 폴백

python 종목분석.py 종목명  ← 단일 종목 재무제표·현재가 수집, 수동 실행
    ├── DART Open API로 최근 3~4개년 사업보고서 + 올해 진행 분기/반기보고서 수집
    ├── KIS Open API(UN, KRX+NXT 통합)로 현재가·시가총액·발행주식수 수집
    ├── 종목분석결과/{종목명}_{YYYYMMDD}.json 저장
    ├── MongoDB company_analysis 컬렉션 저장(_id=종목코드, upsert) → 웹앱 "종목 분석" 탭에서
    │     종목명 검색(자동완성)하면 /api/getCompanyOverview가 바로 조회해 표시 (2026-06-25)
    └── Codex가 이 JSON + dart-mcp/웹 검색으로 서술형 리포트(.docx) 작성 (수동, /종목분석
          스킬 참고, 웹앱 표시와는 별개의 산출물)

python 주도주분석.py [YYYY-MM-DD]  ← 뉴스분석.py 다음에 실행(수동, Claude 미사용), 날짜
생략 시 오늘
    ├── MongoDB stock_data의 그날 vol+rate(거래대금·등락률 상위 50, 최대 100개 중복 제거)를
    │     읽어 종목분석.py의 analyze_one()을 일괄 호출(2026-06-27)
    └── 종목별 company_analysis에 저장된 latest_report가 DART 실제 최신 보고서와 같으면
          스킵(이미 최신), 다르면(새 분기/반기/사업보고서 발생) 그 종목만 재분석
```

> ⚠️ **토스증권 API는 IP 허용 목록 기반**이라 Vercel 서버리스 함수(유동 IP)에서 직접 호출하면
> 막힌다. 반면 **KIS(한국투자증권) Open API는 IP 제한이 없음을 확인**(2026-06-20)했고, 캔들 조회는
> `api/candles.js`가 KIS를 직접 호출한다. 토스 호출은 고정 IP인 로컬 `뉴스분석.py`에서만 계속하며,
> 그 결과는 KIS 호출이 실패했을 때만 쓰이는 폴백 캐시다. 자세한 내용·이유는
> [DATA_PIPELINE.md](DATA_PIPELINE.md) 참고.

  이제 아무 종목이나 KIS로 실시간 조회됨 ([DATA_PIPELINE.md](DATA_PIPELINE.md)).
